## 🎯 新增内容

### 1. 原始组级数据（带ADQ_ID）

**目录**：`submit/original_data_with_ADQ_ID/`
**文件数**：7个（6个CSV + 1个README）

#### 包含文件：
- `control_All_ROI_Summary.csv` (17 KB)
- `mci_All_ROI_Summary.csv` (20 KB)
- `ad_All_ROI_Summary.csv` (20 KB)
- `control_All_Events.csv` (179 KB)
- `mci_All_Events.csv` (435 KB)
- `ad_All_Events.csv` (548 KB)
- `README_ORIGINAL_DATA.txt` (说明文档)

#### 用途：
✅ **数据追溯**：保留完整的ADQ_ID（如m06q3），便于回溯数据来源
✅ **验证合并**：可验证roi_summary_long.csv和events_long.csv的合并正确性
✅ **回答审稿人**：当被问到"m06为什么出现25次"时，可以清晰说明是5个任务×5个ROI

#### ADQ_ID格式：
```
n1q1  = 受试者n01，Task Q1
m06q3 = 受试者m06，Task Q3
a15q5 = 受试者a15（对应ad17），Task Q5
```

---

### 2. ROI字典与映射

**文件**：
- `submit/ROI_Dictionary.txt` (3.7 KB) - 人类可读
- `submit/ROI_Mapping.csv` (485 B) - 机器可读

#### 内容：
- 各任务（Q1-Q5）的完整ROI列表
- ROI类型定义（KW/INST/BG）
- ROI统计摘要

#### 用途：
✅ **可复现性**：明确每个Task包含哪些具体ROI
✅ **透明度**：说明ROI_type的分类规则
✅ **验证性**：任何ROI统计都可追溯到具体ROI名称

#### ROI类型：
- **KW (Keywords)**：关键词区域（如时间词汇、地点名称）
- **INST (Instructions)**：指令区域（任务说明和操作提示）
- **BG (Background)**：背景区域（用于检测注意力分散）

---

### 3. Figure 3重绘材料

**文件**：
- `figure3_data_participant_task_level.csv` (14 KB) - 用于统计分析
- `figure3_data_group_task_level.csv` (1.1 KB) - 用于绘图
- `figure3_plot_script.R` (1.7 KB) - R绘图脚本
- `figure3_plot_script.py` (2.2 KB) - Python绘图脚本
- `README_FIGURE3.txt` (说明文档)

#### 核心修正：
❌ **错误**：Saccade speed / faster/slower saccades
✅ **正确**：**Saccade amplitude (deg)** / larger/smaller amplitudes

#### 用途：
✅ **术语统一**：与Table 6使用相同的"amplitude"术语
✅ **可复现**：提供R和Python绘图代码
✅ **数据透明**：从events_long.csv可追溯到每个数据点

#### 使用方法：
```bash
# R用户
cd submit
Rscript figure3_plot_script.R

# Python用户
cd submit
python3 figure3_plot_script.py
```

## 🎖️ 审稿抗打能力提升

### v1.0存在的潜在风险：

1. **数据追溯性不足**

   - 问题：Table 6用"amplitude"，Figure 3用"speed"，术语冲突
   - 解决：✅ 提供重绘脚本，统一为"amplitude"
