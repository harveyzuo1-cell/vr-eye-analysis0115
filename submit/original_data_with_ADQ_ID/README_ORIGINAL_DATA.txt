================================================================================
原始组级数据文件说明（带ADQ_ID）
================================================================================

目的：
这些文件保留了原始的ADQ_ID（Analysis Data Query ID），用于追溯数据来源。
当审稿人询问特定样本（如m06）为什么出现多次时，可以通过ADQ_ID回溯。

文件清单：
--------------------------------------------------------------------------------

ROI Summary（注视指标汇总）：
  - control_All_ROI_Summary.csv  (Control组，20人)
  - mci_All_ROI_Summary.csv      (MCI组，20人)
  - ad_All_ROI_Summary.csv       (AD组，20人)

Events（眼动事件详细数据）：
  - control_All_Events.csv       (Control组)
  - mci_All_Events.csv           (MCI组)
  - ad_All_Events.csv            (AD组)

ADQ_ID格式：
--------------------------------------------------------------------------------
ADQ_ID = ParticipantID + Task
示例：
  - n1q1  = 受试者n01，Task Q1
  - m06q3 = 受试者m06，Task Q3
  - a15q5 = 受试者a15（对应ad17），Task Q5

与提交数据的关系：
--------------------------------------------------------------------------------
1. roi_summary_long.csv = 合并这3个ROI Summary文件后的long格式
2. events_long.csv = 合并这3个Events文件并筛选无效数据后的结果
3. participants_master.csv = 从subjects.csv和MMSE文件提取的受试者级别数据

数据追溯示例：
--------------------------------------------------------------------------------
如果审稿人问："m06在roi_summary_long.csv中出现了25次，为什么？"

回答步骤：
1. 检查mci_All_ROI_Summary.csv中以m06开头的ADQ_ID
2. 发现m06q1, m06q2, m06q3, m06q4, m06q5共5个任务
3. 每个任务约5个ROI，5×5=25条记录
4. 这是正常的重复测量设计（5 tasks × ~5 ROIs per task）

================================================================================
注意事项：
================================================================================
- 这些文件未经过筛选，包含所有原始数据（包括Duration=0的占位符）
- AD组ID映射：文件中的a01-a20对应subjects.csv中的ad03-ad22
- 用于数据验证和追溯，不用于直接分析
